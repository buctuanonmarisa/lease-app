# LeaseFlow — AI Tenant Screening Platform

LeaseFlow is a full-stack rental application platform designed for the Philippine market. It automates tenant screening from initial application through document verification, AI analysis, and final admin decision — reducing screening time from days to hours.

---

## Purpose

Property managers receive a structured, data-rich view of every applicant (credit score, income ratio, employment status, landlord sentiment) generated by an AI layer the moment a tenant submits their application. Tenants can track their application status in real time and fix flagged documents without contacting the office.

---

## User Flows

### Tenant Application Flow

```
/apply  →  /apply/documents  →  /apply/review  →  /apply/confirmed/[id]
```

1. **Apply** (`/apply`) — Fill out personal info, select a rental unit, and enter gross monthly income. A live income-to-rent ratio check enforces the 2.5× minimum.

2. **Documents** (`/apply/documents`) — Upload 7 required documents:
   - 3× Pay stubs
   - Certificate of Employment
   - Credit Report
   - Government-Issued ID
   - Landlord Reference Letter

   A progress bar tracks completion. Files are validated for type (PDF/JPG/PNG) and size (max 10 MB).

3. **Review & Submit** (`/apply/review`) — Preview all entered data and uploaded documents, then submit. The server generates an AI analysis and assigns an Application ID (`APP-YYYY-XXXX`). A confirmation email is sent automatically.

4. **Confirmation** (`/apply/confirmed/[id]`) — Displays the Application ID and outlines next steps.

---

### Application Tracking Flow

```
/status  →  /status/[id]?email=
```

1. **Lookup** (`/status`) — Enter an Application ID and email address to find an application.

2. **Status Detail** (`/status/[id]`) — Displays the current status and AI screening criteria:

   | Status | Meaning |
   |---|---|
   | **Processing** | AI is analyzing documents (24–48 hrs) |
   | **Review** | Passed AI checks; awaiting admin decision (1–2 days) |
   | **Mismatch** | Documents flagged; tenant must re-upload |
   | **Approved** | Application approved; next steps shown |
   | **Rejected** | Application rejected with reason |

   Tenants with a **Mismatch** status can re-upload flagged documents directly from this page via a modal.

---

### Admin Flow

```
/admin/login  →  /admin/dashboard  →  /admin/applications/[id]
```

1. **Login** (`/admin/login`) — JWT-based authentication. Roles: `superadmin` and `admin`.

2. **Dashboard** (`/admin/dashboard`) — Overview of all applications with stats, search, and filter tabs (All / Review / Mismatch / Processing / Approved / Rejected). Color-coded credit scores and income ratios make at-a-glance triage easy.

3. **Application Detail** (`/admin/applications/[id]`) — Split-screen view:
   - **Left** — AI screening results: credit score, income ratio, employment verification, landlord sentiment, document completeness.
   - **Right** — Document viewer with tabs for each of the 7 required files.
   - Admin enters required decision notes, then clicks **Approve** or **Reject**. An automated email is sent to the applicant immediately.
   - Prev/Next navigation moves between applications without returning to the dashboard.

## AI Screening

The AI layer runs automatically on submission and evaluates:

- **Credit Score** — Derived from income ratio + a randomized component (range 490–850). Green ≥ 680, yellow ≥ 600, red < 600.
- **Income Ratio** — Gross monthly income ÷ monthly rent. Must meet 2.5× minimum.
- **Employment Consistency** — Cross-checks pay stubs against employment certificate.
- **Landlord Sentiment** — Analyzes the landlord reference letter (Positive / Neutral / Negative).
- **Document Completeness** — Flags missing or mismatched documents.

Auto-assigned statuses based on AI output:

| AI Result | Initial Status |
|---|---|
| All checks pass | `processing` (admin still decides) |
| Partial flags | `review` |
| Document discrepancy | `mismatch` |
| Credit + income both fail | `rejected` |

## Setup

```bash
cd lease-app
npm install
npx prisma migrate dev --name init
node prisma/seed.js
npm run dev
```

Create a `.env.local` file:

```env
DATABASE_URL=file:./dev.db
JWT_SECRET=your-long-random-secret
RESEND_API_KEY=your-resend-key
APP_URL=http://localhost:3000
EMAIL_FROM=noreply@yourdomain.com
```
